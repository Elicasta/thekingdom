# The Kingdom

A single-lesson static web app for **Lesson 2: When the Kingdom Falls**.

## Included

- 23-slide presentation with keyboard, swipe, overview, and full-screen controls
- Leader teaching guide
- Three anonymous live polls backed by Supabase
- Reflection questions with local autosave
- No login, names, analytics, or remote reflection storage
- Installable offline shell through a service worker

## Data behavior

### Polls

When Supabase is configured, each browser sends:

- the poll ID
- the selected option
- a random device UUID generated by the browser

The UUID is not a name or account. It lets the same browser update its current answer instead of adding a second vote. Results returned to the app contain aggregate counts only.

When Supabase is not configured or cannot be reached, the choice still stays on that phone and the app clearly reports that live totals are unavailable.

No-login polling cannot prove one human equals one vote. A determined person can clear browser storage or use another browser. For a one-room youth lesson, this is the cleanest tradeoff. Real identity enforcement would require auth or event-issued access codes.

### Reflections

Reflection answers use browser `localStorage` only. They never leave the device. Clearing browser data, using private browsing, or switching phones will remove or hide them.

## Supabase setup

1. Create a Supabase project.
2. In **SQL Editor**, run `supabase/setup.sql`.
3. In **Project Settings → API**, copy the project URL and the **publishable** key.
4. Put both values in `config.js`.
5. Deploy and test from two browsers or phones.

Never put a service-role key, secret key, or database password in browser files.

The in-app clear button removes that phone’s votes from live totals when Supabase is reachable. To clear every participant’s totals before a new session, run `supabase/reset-polls.sql` manually.

## Run locally

```bash
python3 -m http.server 8080
```

Open `http://localhost:8080`.

`file://` mode is not supported because service workers and browser network rules require a web origin.

## Deploy

The folder is ready for Vercel or another static host. Upload the folder as-is after filling `config.js`.

## Slide controls

- Right arrow, Page Down, or Space: next slide
- Left arrow or Page Up: previous slide
- Home: first slide
- End: last slide
- F: full screen
- Swipe left or right on touch screens
